時間の経過とともに、エージェントセッションにはコンテキストが蓄積され、レスポンスが遅くなったり混乱を招くことがあります。`tetora session` コマンドでこれを管理できます。

## 新しいセッションを作成

```bash
tetora session new --agent kokuyou
```

指定したエージェントの新しいセッションを開始し、以前のセッションは履歴に保存されます。

## 使用するタイミング

- エージェントが混乱しているか、古いコンテキストを参照している場合
- 以前の作業とは無関係な新しいタスクを開始する場合
- コードベースの大幅な変更後、以前のコンテキストが無効になった場合
- エージェントのパフォーマンスベンチマークのためにクリーンな状態が必要な場合

## セッション管理

```bash
tetora session list                    # 全セッションを一覧表示
tetora session show <id>               # セッション詳細を表示
tetora session switch <id>             # 以前のセッションに切り替え
tetora session delete <id>             # セッションを削除
```

## ヒント

- セッションはエージェントごと — あるエージェントの新規セッション作成は他のエージェントに影響しません
- 以前のセッションは `tetora history` で検索可能です
- 設定で `session.auto_rotate: true` を設定すると、毎日自動的に新しいセッションが作成されます
